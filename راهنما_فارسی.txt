==================================================
راهنمای کامل پروژه سیستم مدیریت پیام (فارسی)
==================================================

این فایل برای این نوشته شده که بدون نیاز به دانستن برنامه‌نویسی،
بفهمید پروژه از چه بخش‌هایی تشکیل شده، هر فایل چه کاری می‌کند،
و مهم‌تر از همه: کجا باید رمز/نام کاربری را عوض کنید و کجا دیتابیس
وصل می‌شود.


==================================================
۱) ساختار کلی پروژه
==================================================

Project_Message_updated/
│
├── backend/                      ← همه‌ی کدهای سمت سرور (PHP) اینجاست
│   │
│   ├── config/
│   │   ├── db.php                ← تنظیمات اتصال به دیتابیس (رمز/یوزر MySQL اینجاست)
│   │   └── session.php           ← بررسی اینکه آیا ادمین وارد شده یا نه
│   │
│   ├── api/
│   │   ├── auth.php              ← ورود / خروج / بررسی وضعیت ورود ادمین
│   │   ├── messages.php          ← لیست کردن، ثبت و حذف پیام‌ها
│   │   └── categories.php        ← لیست کردن، ساخت و حذف دسته‌بندی‌ها
│   │
│   └── database.sql              ← اسکریپت ساخت دیتابیس و جدول‌ها + کاربر ادمین پیش‌فرض
│
└── frontend/                     ← صفحات HTML که کاربر در مرورگر می‌بیند
    │
    ├── admin/
    │   ├── login.html             ← صفحه ورود ادمین (نام کاربری + رمز)
    │   └── index.html             ← پنل مدیریت (نوشتن/حذف پیام، ساخت دسته‌بندی)
    │
    └── user/
        └── index.html             ← تابلوی عمومی که همه می‌توانند پیام‌ها را ببینند


نحوه کار کلی سیستم:
  کاربر عادی  →  frontend/user/index.html  →  می‌خواند پیام‌ها را (فقط نمایش)
  ادمین       →  frontend/admin/login.html →  وارد می‌شود
              →  frontend/admin/index.html →  پیام و دسته‌بندی اضافه/حذف می‌کند

هر دو صفحه‌ی frontend با فایل‌های backend/api/*.php صحبت می‌کنند،
و آن فایل‌ها هم با دیتابیس (MySQL) صحبت می‌کنند.


==================================================
۲) کجا باید تنظیمات دیتابیس (رمز/یوزر MySQL) را عوض کنم؟
==================================================

مسیر فایل:  backend/config/db.php

داخل این فایل ۴ خط مهم هست:

    define('DB_HOST', 'localhost');    → آدرس سرور دیتابیس
    define('DB_NAME', 'messages_db');  → نام دیتابیس
    define('DB_USER', 'root');         → نام کاربری MySQL
    define('DB_PASS', '');             → رمز عبور MySQL

اگر پروژه را روی هاست/سروری غیر از کامپیوتر خودتان (XAMPP/Laragon)
آپلود می‌کنید، این ۴ مقدار را باید با اطلاعاتی که شرکت هاستینگ یا
phpMyAdmin در اختیارتان گذاشته، جایگزین کنید. برای مثال:

    define('DB_HOST', 'localhost');
    define('DB_NAME', 'user_messages_db');
    define('DB_USER', 'user_dbuser');
    define('DB_PASS', 'RamzeMaXusi123!');

⚠️ توجه: این رمز، «رمز دیتابیس» است، نه رمز ورود به پنل ادمین سایت.
این دو کاملاً با هم فرق دارند (توضیح رمز پنل ادمین در بخش ۳).

نکته فنی: در فایل db.php، تابعی به نام ()getDB وجود دارد که واقعاً
با استفاده از همین ۴ مقدار به دیتابیس وصل می‌شود. تمام فایل‌های
backend/api/messages.php ،auth.php و categories.php با صدا زدن
()getDB یک اتصال دیتابیس می‌گیرند و سپس با دستورات SQL
(SELECT / INSERT / DELETE) داده‌ها را می‌خوانند یا تغییر می‌دهند.
همه‌ی این نقطه‌ها در کد با کامنت "🔌 اتصال به دیتابیس" مشخص شده‌اند.


==================================================
۳) کجا باید نام کاربری/رمز عبور ورود به پنل ادمین را عوض کنم؟
==================================================

این رمز، برخلاف رمز دیتابیس، داخل هیچ فایل PHP نوشته نشده؛
داخل خودِ دیتابیس، در جدولی به نام admins ذخیره می‌شود.

مقدار پیش‌فرضی که با اجرای backend/database.sql ساخته می‌شود:

    نام کاربری: admin
    رمز عبور:   admin123

برای تغییر آن، دو روش دارید:

  روش ۱ (ساده‌تر - از طریق phpMyAdmin):
    ۱. وارد phpMyAdmin شوید و دیتابیس messages_db را باز کنید.
    ۲. جدول admins را باز کنید.
    ۳. یک صفحه‌ی کوچک PHP بسازید (یا از ابزار آنلاین امن استفاده کنید)
       که خروجی تابع زیر را برایتان بسازد:

           password_hash('رمز-جدید-من', PASSWORD_BCRYPT)

       این تابع یک رشته‌ی رمزنگاری‌شده (هش) تولید می‌کند، مثلاً:
           $2y$10$abcdefghijklmnopqrstuv....

    ۴. آن رشته را در ستون password_hash همان ردیف ادمین جایگزین کنید.
       (هرگز رمز را به صورت متن ساده مثل admin123 داخل این ستون ننویسید،
       چون سیستم برای مقایسه‌ی رمز از password_verify استفاده می‌کند
       که فقط با هش کار می‌کند، نه با متن ساده.)

  روش ۲ (از طریق کد، برای کسی که کمی PHP بلد است):
    یک فایل موقت به نام make_hash.php در پوشه backend بسازید با محتوای:

        <?php
        echo password_hash('رمز-جدید-من', PASSWORD_BCRYPT);

    آن را در مرورگر باز کنید، رشته‌ی خروجی را کپی کنید،
    در phpMyAdmin در ستون password_hash جایگزین کنید،
    و در پایان، فایل make_hash.php را حتماً حذف کنید (به‌خاطر امنیت).

نقطه‌ای که در کد این مقایسه انجام می‌شود:
  فایل: backend/api/auth.php
  خط مربوطه: password_verify($password, $admin['password_hash'])
  (در کد با کامنت توضیح داده شده است)


==================================================
۴) خلاصه فایل به فایل (چه کاری می‌کند)
==================================================

backend/config/db.php
  تنظیمات اتصال به MySQL (رمز/یوزر) + تابع getDB() که اتصال را می‌سازد.

backend/config/session.php
  مدیریت نشست (Session) مرورگر و بررسی اینکه آیا کاربر فعلی
  به عنوان ادمین وارد شده یا نه. تابع requireAdmin() جلوی
  عملیات‌های حساس (ثبت/حذف پیام) را برای کاربران غیرادمین می‌گیرد.

backend/api/auth.php
  آدرس: /api/auth.php
  ?action=login   (POST) → بررسی نام کاربری/رمز و ورود
  ?action=logout  (POST) → خروج از حساب
  ?action=status  (GET)  → آیا الان وارد شده‌ایم یا نه

backend/api/messages.php
  آدرس: /api/messages.php
  GET    → گرفتن لیست پیام‌ها (با امکان فیلتر بر اساس دسته‌بندی)
  POST   → ثبت پیام جدید (فقط ادمین)
  DELETE → حذف پیام (فقط ادمین)

backend/api/categories.php
  آدرس: /api/categories.php
  GET    → گرفتن لیست دسته‌بندی‌ها (همراه تعداد پیام هرکدام)
  POST   → ساخت دسته‌بندی جدید (فقط ادمین)
  DELETE → حذف دسته‌بندی (فقط ادمین)

backend/database.sql
  اسکریپت SQL که دیتابیس و جدول‌های categories، messages و admins
  را می‌سازد و چند داده‌ی نمونه + کاربر ادمین پیش‌فرض درج می‌کند.

frontend/admin/login.html
  فرم ورود ادمین. اطلاعات را به auth.php?action=login می‌فرستد.

frontend/admin/index.html
  پنل مدیریت: نوشتن پیام جدید، انتخاب/ساخت دسته‌بندی، دیدن لیست
  پیام‌ها و حذف آن‌ها. قبل از نمایش، وضعیت ورود را از auth.php چک می‌کند.

frontend/user/index.html
  تابلوی عمومی؛ فقط نمایش پیام‌ها و دسته‌بندی‌ها، بدون نیاز به ورود.


==================================================
۵) درباره تگ‌های HTML (برای کسی که با HTML آشنا نیست)
==================================================

در فایل‌های .html این پروژه، هر بخش با کامنت (توضیح) مشخص شده که
داخل <!-- ... --> نوشته می‌شود و در مرورگر دیده نمی‌شود، فقط در کد
قابل مشاهده است. چند نمونه مهم:

  <html lang="fa" dir="rtl">   → می‌گوید زبان صفحه فارسی و از راست به چپ است
  <head> ... </head>            → تنظیمات پشت‌صحنه صفحه (عنوان تب، استایل)
  <meta charset="UTF-8">        → کدگذاری که فارسی درست نمایش داده شود
  <style> ... </style>          → ظاهر و رنگ‌بندی صفحه (CSS)
  <body> ... </body>            → محتوای واقعی صفحه که کاربر می‌بیند
  <div class="..."> ... </div>  → یک جعبه/بخش برای گروه‌بندی محتوا
  <form id="login-form">        → فرم ورود اطلاعات (مثل فرم لاگین)
  <input type="password">       → فیلد وارد کردن رمز عبور
  <button onclick="...">        → دکمه‌ای که با کلیک یک تابع جاوااسکریپت را اجرا می‌کند
  <script> ... </script>        → کدهای جاوااسکریپت که با API (فایل‌های PHP) صحبت می‌کنند

هر کدام از این‌ها در فایل‌های HTML با کامنت فارسی مشخص شده‌اند تا
مشخص باشد هر تگ برای چه بخشی از صفحه است.


==================================================
۶) نکته مهم: خطای "خطا در اتصال به سرور"
==================================================

این خطا وقتی نمایش داده می‌شود که فایل‌های HTML در frontend نمی‌توانند
به فایل‌های API در backend وصل شوند — معمولاً به‌خاطر مسیر اشتباه.

ساختار پروژه این‌طور است:

    پوشه پروژه/
      backend/api/messages.php
      frontend/user/index.html

از داخل frontend/user/index.html، برای رسیدن به backend/api باید
دو پوشه بالا رفت (../../) و بعد وارد backend/api شد:

    مسیر درست: ../../backend/api/messages.php

(این پروژه از قبل با همین مسیر درست تنظیم شده. اگر خودتان فایل‌ها را
جابه‌جا کردید یا ساختار پوشه‌ها را عوض کردید، مسیرها را در این فایل‌ها
بررسی کنید:
  frontend/user/index.html
  frontend/admin/index.html
  frontend/admin/login.html
داخل تگ <script> در انتهای هرکدام، ثابت‌های MESSAGES_API،
CATEGORIES_API و AUTH_API را پیدا کنید.)

⚠️ همچنین حتماً مطمئن شوید:
  - در XAMPP Control Panel هر دو سرویس Apache و MySQL روشن (Running) باشند.
  - آدرس مرورگر با http://localhost/ شروع شود، نه file:///
  - پوشه backend و frontend کنار هم و داخل همان پوشه پروژه در htdocs باشند
    (یکی را جابه‌جا یا حذف نکنید).


==================================================
۷) راه‌اندازی سریع (چک‌لیست)
==================================================

۱. فایل backend/database.sql را در phpMyAdmin (یا MySQL خط فرمان) اجرا کنید
   تا دیتابیس و جدول‌ها ساخته شوند.
۲. فایل backend/config/db.php را باز کنید و DB_USER/DB_PASS را با
   اطلاعات واقعی MySQL خودتان جایگزین کنید (بخش ۲ همین راهنما).
۳. پروژه را روی یک سرور PHP اجرا کنید (مثلاً با دستور php -S در پوشه پروژه،
   یا آپلود روی هاست با پشتیبانی PHP و MySQL).
۴. برای دیدن تابلوی عمومی: frontend/user/index.html
۵. برای ورود به پنل ادمین: frontend/admin/login.html
   (نام کاربری: admin ، رمز: admin123 — حتماً بعداً عوض کنید، بخش ۳)
